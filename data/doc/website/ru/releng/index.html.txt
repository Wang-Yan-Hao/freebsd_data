Документация по подготовке релизов

-   Подготовка релизов FreeBSD
    Описывает подход, используемый командой подготовки релизов FreeBSD
    для создания релизов, качество которых соответствует готовности к
    промышленной эксплуатации операционной системы FreeBSD. В этом
    документе рассказано об инструментах, доступных тому, кто
    заинтересован в создании собственных релизов FreeBSD для
    использования в корпоративной среде или в коммерческих целях.

Команда подготовки релизов

Основная группа подготовки релизов отвечает за подтверждение запросов
MFC в периоды замораживания кода, создание планов выпуска и остальные
работы, описанные в наших обязанностях.

Основная группа подготовки релизов (Primary RE Team) re@FreeBSD.org) :
Glen Barber <gjb@FreeBSD.org>, Konstantin Belousov <kib@FreeBSD.org>,
Marc Fonvieille <blackend@FreeBSD.org>, Mike Karels
<karels@FreeBSD.org>, Xin Li <delphij@FreeBSD.org>, Colin Percival
<cperciva@FreeBSD.org> формирует основную группу подготовки, принимающую
решения.

Группа построения релизов отвечает за построение и создание
дистрибутивов релизов FreeBSD для различных поддерживаемых платформ.

Группа построения релизов(re-builders@FreeBSD.org) : Marcel Moolenaar
<marcel@FreeBSD.org>, Yoshihiro Takahashi <nyan@FreeBSD.org>, Nathan
Whitehorn <nwhitehorn@FreeBSD.org>

Пакеты сторонних производителей в Коллекции Портов управляются группой
portmgr@. Кроме прочих обязанностей, ответственные за порты отвечают за
гладкую работу кластера портов в процессе создания двоичных пакаджей.

Создатели пакаджей (portmgr@FreeBSD.org) : Mathieu Arnold

Старые релизы

Проект FreeBSD не поддерживает централизованный исторический архив ISO
образов старых релизов но большинство из них доступно по этому адресу:
ftp://ftp-archive.FreeBSD.org/pub/FreeBSD-Archive/old-releases/.

Если вы не можете найти зеркало FTP, на котором всё ещё хранится искомый
релиз, то можете написать поставщикам компакт-дисков, есть ли у них
старые релизы.
